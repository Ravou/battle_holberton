Règles du PLD Battle

Le PLD Battle est un jeu pédagogique inspiré du pair programming, où deux groupes d'étudiants s’affrontent sur des tâches de programmation.

Déroulement

  • Les étudiants sont répartis en 2 groupes (A et B).
  • À chaque tâche, un codeur différent est désigné pour écrire le code.
  • Chaque tâche est limitée dans le temps (5 minutes).
  • Chaque solution passe dans un checker qui vérifie :
    • Le bon fonctionnement du code (sorties attendues).
    • Le respect du style (Betty coding style).
    • Le succès de la compilation simulée.
  • Le groupe qui obtient le plus de points remporte la tâche.

Système de points

  • Chaque test réussi = 1 point.
  • Certains cas spéciaux = 2 points.
  • Le total détermine le gagnant de la tâche.

Victoire finale

L’équipe qui a gagné le plus de tâches est déclarée vainqueur du PLD Battle.
